Ship plan

How I'd ship Cortex

The autonomy dial per segment, where it sits on the trust ladder, how it deploys, what it returns, and how I keep it safe.

01 · Dial

Autonomy per segment

- Seasoned PM → Bounded-autonomous → trusts Cortex to assemble + queue; spot-checks, doesn't read each
- New eng lead → Supervised → approves every story-prep action at the checkpoint
- Exec stakeholder → Assisted → sees suggestions, acts manually in their own voice
02 · Ladder

Rung + eval gate

- Current rung: Supervised (every output waits for approval at the HITL checkpoint; with no publish tool, it structurally can't self-promote).
- Eval gate to climb to Bounded-autonomous: critic false-pass ≤2% (≤1% on the date-promise/false-green subset) AND confidentiality-leak rate = 0 AND human-override <2% — held over a rolling 30-day / ≥50-run window on a ≥24-case labeled harness.
- Hard stop: miss any one → stay Supervised (evidence, not calendar); #8 company-wide post never crosses short of full autonomy.
03 · Deploy

Where it runs & who owns it

- Runtime: GitHub Actions scheduled cron (Monday) + workflow_dispatch for manual/replay; ANTHROPIC_API_KEY as an encrypted secret, bounds pinned as workflow env. Scheduled, not hook-triggered — the load is one run a week, not bursty.
- On-call owner: named PM owner at Supervised, where nothing pages because nothing posts; named eng owner of agent.py for pipeline failures (bad key, connector down). Not "the team." At Bounded this splits: eng halts machinery, PM adjudicates content.
- Rollback: disable the cron (off is instant — no write tool means no partial state to unwind), revert prompt/bounds to a known-good SHA, disable propose_stories to degrade to draft-only, or drop the dial a rung.
Monitoring: live critic pass %, escalation rate, cost-to-serve vs. the $0.50 cap, and trust incidents (leaks / false-greens = 0).
04 · ROI

Value, not activity

- Outcome: draft-accepted-without-substantive-edit rate (≥90% target), and ~30–45 min/week of PM assembly saved — both measured against the rung-1 shadow-mode baseline, not asserted.
- Cost-to-serve: fully-loaded $ per accepted update (model + tools + retries + human review) ≈ $5–8/run — ~97–99% of it the ~2-min human review; model+tools ~$0.05–0.18.
- Trust incidents: leaks / false-greens / unflagged bad dates = 0; any single one pauses autonomy expansion.
05 · Governance

How I keep it safe

- Compliance: embargoed roadmap items do enter the prompt (Cortex must read them to know what to exclude) — the control is they never reach the output, enforced by critic checks #3/#4 + the roadmap CONFIDENTIAL/warning field + norms. At Bounded this gains a dedicated pre-publish confidentiality scan, since the inline human is gone. No PII scrubber today (corpus is local reads); that's a connector-milestone control.
Safety: no world-acting tool exists — no post/create/merge — so "post it" is unsatisfiable by construction (a missing tool is the strongest bound); over-cap story batches are rejected, not split (cap 10); kill switch = Ctrl-C/SIGTERM + cost-cap auto-halt; JIT per-run read-only token lands with the first live connector.
- Reliability: cost cap $0.50 + iteration cap 8 + revision cap 2, each enforced outside the model; escalate-on-stuck (missing data → escalate, never fabricate); wall-clock timeout specced, ships with the connector.
Strategy: widen one segment at a time (the autonomy dial), evidence not calendar; build the ≥24-case labeled harness, then climb on higher-volume fleet tasks (activity triage, first-draft grooming) — the exec update may stay deliberately forever-Supervised.
Build insights

What I learned

The honest three: where it hurt, what clicked, and the moment something changed.

😵 Friction

The hard part was never the code — it was design under uncertainty. Many data sources × different sensitivity levels means the real work is up front: defining bounds, anticipating failure modes, engineering so a confused run can't do something irreversible.

🧠 Learning

Start small, expand on evidence. Few tasks, few sources, all reads — a blast radius small enough to reason about completely. Widen scope only after the smaller version proves it behaves.

💡 Aha

Pulling confidentiality out as its own decision — instead of averaging it into "what context is relevant" — is what made the real risk visible. And visible is the precondition for controllable.
