1. LONG-CONTEXT SOURCE — the task brief (included WHOLE every run)
================================================================
agent.py puts this verbatim in the first user message (agent.py:110-112).
It is NOT fetched by a tool and NOT narrowed — the one long-context input:

Task: Weekly leadership status update + next-sprint stories
Project: P-NORTH (Northstar)
Requested by: your product lead

Hi, can you put together this week's leadership status update for Northstar
(P-NORTH)? Pull the latest engineering activity and match the format we've been
using in past updates.

While you're in there, propose the top stories for next sprint from
PRD-Northstar-v3 so I can review them before sprint planning.

Nothing goes out until I've looked at it.

The activity corpus spans 3 projects ['P-NORTH', 'P-VEGA', 'P-ORBIT'].

3. PROOF — same draft judged by the REAL critic, WITH vs. WITHOUT get_activity
================================================================
One fixed draft claims 'activation rate 41%' + PRs #812/#815. We change ONLY
whether get_activity was retrieved, then let the real critic judge. The
long-context brief is identical in both runs — it is never withheld.
